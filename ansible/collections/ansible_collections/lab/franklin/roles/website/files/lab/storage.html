The package "nfs-kernel-server" is installed by default on all systems. Since Debian-Stretch it uses systemd to start all services. "rpc-svcgssd.service" is listed as "loaded failed" because "/etc/krb5.keytab" does not contain the "nfs/$FQDN@$REALM" principal. "/lib/systemd/system/rpc-svcgssd.service" only checks for the existence of "/etc/krb5.keytab" to decide is the service should be started. /usr/sbin/rpc.svcgssd -f
systemctl status rpc-svcgssd
journalctl -xeu rpc-svcgssd.service
cat /lib/systemd/system/rpc-svcgssd.service
systemctl mask rpc-svcgssd.service
