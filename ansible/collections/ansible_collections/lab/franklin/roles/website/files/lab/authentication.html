Lab Authentication

SSH Configuration

Host *.lab.bitsmasher.net GSSAPIAuthentication yes GSSAPIDelegateCredentials yes CanonicalizeHostname yes CanonicalDomains lab.bitsmasher.net CanonicalizeMaxDots 3 CanonicalizeFallbackLocal yes

Kerberos Client Setup

sudo apt install krb5-user krb5-config libpam-krb5 libpam-ccreds
sudo apt install ldap-utils

Now set up the keytab for the host. Log in to the host to Create /etc/krb5.keytab file in place. Must be done as root user (not sudo)

sudo -i
kinit root/admin
kadmin -p root/admin
# add host and ldap principals
addprinc -randkey host/head1.lab.bitsmasher.net
addprinc -randkey ldap/head1.lab.bitsmasher.net
ktadd host/head1.lab.bitsmasher.net ldap/head1.lab.bitsmasher.net
(quit)
klist -ke /etc/krb5.keytab
 kinit franklin -X X509_user_identity=FILE:~/.ssh/franklin-cert.pem,~/.ssh/franklin-key.pem
KRB5_TRACE=/dev/stdout ssh

Troubleshoot KDC

kinit -p root/admin klist -ke /etc/krb5.keytab
env KRB5_TRACE=/dev/stdout kadmin -p root/admin@LAB.BITSMASHER.NET

Troubleshoot Client

  • Check sudo hostnamectl
  • Check keytab "sudo klist -ke /etc/krb5.keytab"
  • Start sshd in console/debug-mode: mkdir /run/sshd && $(which sshd) -ddd

Kerberos User Setup

Start the kadmin tool

kadmin.local -p admin/admin
ktutil --keytab=/etc/krb5.keytab list

Create the LDAP principal & keytab

addpol -minlength 8 -maxlife 7776000 -history 5 users
getpol users
ank -policy users franklin
listprincs

Preauth: Put the pem files in /var/lib/krb5kdc since this is read/writeable by the KDC.

kinit root/admin
kadmin -q 'addprinc -randkey WELLKNOWN/ANONYMOUS'
